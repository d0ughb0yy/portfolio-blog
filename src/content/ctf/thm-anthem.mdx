## Overview

A Windows Server machine running an Umbraco CMS installation. The key lesson from this box: always check for exposed credentials and remember to enable hidden files when enumerating Windows systems.

## Port Scan

Only two ports were open:
- Port 80 - HTTP (Microsoft HTTPAPI)
- Port 3389 - RDP

## Initial Access

The robots.txt file contained an interesting entry: `UmbracoIsTheBest!`

This turned out to be the password for the Umbraco admin panel. Using the information gathered from the blog posts on the site (one mentioned "Solomon Grundy" hinting at the admin, another leaked an email pattern), I determined the admin email was `SG@anthem.com`.

With these credentials, I logged into the Umbraco admin panel and found I could reuse the same password for RDP access.

## Privilege Escalation

The path to Administrator involved a misconfigured backup folder. After taking ownership of the backup file, I found the Administrator password stored within.

## Key Takeaways

- Enumerate carefully - sometimes the answers are in plain sight
- When enumerating Windows machines, always enable hidden files and directories
- Credential reuse is still a prevalent issue in internal networks
